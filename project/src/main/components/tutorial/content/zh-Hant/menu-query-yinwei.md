# 查音位

這一頁是從「現代音值 / 音位表現」反推中古來源的入口。它比查字、查中古都更分析型：你先告訴系統某類音位條件，再看各地哪些中古類別會落到這裡。

## 你在這一頁會看到什麼

- 第一行仍先選聲母、韻母、聲調。
- 可選鍵名，頁面中間會用一句提示文字說明「你現在正在分析哪一組鍵」。
- 真正輸入音位內容的是下方的音位選擇 / 輸入區。
- 一樣可以配合地點 / 分區一起限制範圍。
- 按下運行後，會跳到結果頁，而不是留在本頁展開。

## 最適合新手的操作順序

1. 先決定你要分析的是聲母、韻母還是聲調。
2. 再選鍵名，讓頁面先知道你想分析哪一個中古維度。
3. 看中間那句提示文字，確認自己現在選的分析方向沒有選錯。
4. 到下方音位輸入區填入或選擇實際音位內容。
5. 如有需要，再補地點 / 分區，最後按運行。

## 這一頁有哪些實際規則

- 音位輸入區沒有完成有效輸入前，運行按鈕會保持不可用。
- 鍵名和音位輸入是一起工作的：你換了分析維度，原本的理解方式也要跟著變。
- 與查中古一樣，這頁也有排除下拉；排除條件會進入實際查詢，不只是畫面提示。
- 執行時會把音位輸入、已選鍵名、地點 / 分區和當前字表一起送往結果頁處理。

## 新手怎麼理解這頁

- 如果說「查中古」是由中古條件找現代結果，那「查音位」就是由現代表現追中古來源。
- 這一頁比較適合回答：某個音值為什麼會出現、它在不同地點可能對應哪些來源。
- 第一次用時不要同時選太多鍵名，先讓問題單純一點，才看得出結果邏輯。

## 地點 / 分區怎麼配合使用

- 音位反查常常結果面很廣，先選幾個地點會更容易理解。
- 如果你是在做區域性觀察，可以先用分區；如果只是核對一兩個點，直接輸入地點更快。

## 新手最常見的卡點

- 按鈕是灰的：多半是下方音位輸入還沒形成有效條件。
- 結果看起來很亂：通常是分析維度選太多，或地理範圍太大。
- 不確定自己在查什麼：先回頭看中間的提示文字，它就是這頁當前分析方向的摘要。

## 什麼時候改用別的頁面

- 你只想查某個字怎麼讀：改用「查字」。
- 你已經明確知道中古條件：改用「查中古」。
- 你要做兩組條件的對照，不是單組反查：改用比較類頁面。
