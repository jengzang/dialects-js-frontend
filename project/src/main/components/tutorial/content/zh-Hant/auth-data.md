# 我的數據

「我的數據」是獨立路由 `/auth/data`，但通常是從個人中心的「我的數據」按鈕進來，並把目前用戶名放在 query `username` 裡供頁首顯示。真正決定你能不能操作的，不是這個 query，而是登入狀態。

## 進頁前先知道兩件事

- 如果未登入就直接打開這頁，掛載時會先跳提示，然後直接導回 `/auth`。
- 頁首看到的 `username` 主要是展示用；資料抓取本身呼叫的是 `getAllCustomData()`。

## 你會看到什麼

- 頁首有返回按鈕、標題，以及當前顯示的用戶名 badge。
- 右側會顯示兩個即時統計：總筆數、已選筆數。
- 工具列有四個主要按鈕：批量新增、批量編輯、批量刪除、重新整理。
- 下方是可搜尋、可分頁、可勾選的資料表格。

## 最適合新手的操作順序

1. 先按一次重新整理，確認資料有正常載入。
2. 先看表格欄位與搜尋是否找得到自己的記錄。
3. 只改單筆時，直接按每列右側的編輯。
4. 要一次增修多筆時，再用批量新增或批量編輯。
5. 刪除前先看清楚已選筆數，避免把整頁勾太多。

## 表格與選取有哪些實際規則

- 搜尋框一輸入就會即時篩選，並把頁碼重設到第 1 頁。
- 分頁大小可切成 10 / 20 / 50 / 100；切換後也會回到第 1 頁。
- 表頭全選只會選到「目前這一頁」的可見資料，不是整個資料集。
- 批量編輯、批量刪除按鈕在未勾選任何資料時會是禁用狀態。
- 表格為空時，若有搜尋字串會顯示「無符合結果」類型提示；沒有搜尋時則顯示一般無資料狀態。

## 單筆編輯怎麼做

- 每一列只有一個「編輯」按鈕，沒有單筆刪除按鈕。
- 按下後會打開小型編輯彈窗。
- 在彈窗中修改後按儲存，會直接呼叫 `editCustomData()`。
- 成功後彈窗會關閉，並重新抓整份資料。

## 批量新增怎麼做

- 按「批量新增」會開啟大型彈窗；如果目前沒有任何列，系統會先自動幫你建立 1 個空白列。
- 你可以：
  - 按 `➕` 再加空白列
  - 按清空按鈕刪掉目前所有草稿列
  - 直接把以 Tab 分隔、以換行分筆的資料貼進表格區域
- 貼上時，系統會把每行拆成 7 欄：簡稱、音典分區、經緯度、聲韻調、特徵、值、說明。
- 只有必填欄位完整的列，才會算進可提交筆數。
- 真正送出時如果有效列是 0 筆，會被擋下；超過 50 筆也會被擋下。

## 批量編輯怎麼做

- 先在表格勾選要改的資料，再按「批量編輯」。
- 彈窗會把已選資料複製成可編輯列。
- 只有必填欄位完整的列，儲存按鈕才會有可提交數量。
- 送出前一定會先跳確認框。
- 這個批量編輯的實作不是逐列 patch，而是：
  1. 先用 `created_at` 刪除原資料
  2. 再把有效列當成新資料重新建立
- 所以如果你把某些列改成不完整，它們不會進入有效列，也就不會被重新建立。

## 批量刪除怎麼做

- 先勾選資料，再按批量刪除。
- 送出前會先顯示確認框。
- 確認後會把目前勾選的 `created_at` 一次送去刪除。
- 成功後會清空選取狀態，並重新整理表格。

## 新手最常見的卡點

- 以為返回按鈕會回瀏覽器上一頁：實際上它直接 `router.push('/auth')` 回帳號頁。
- 以為全選會選到所有分頁：其實只選當前頁。
- 以為批量編輯是原地改欄位：實際流程是先刪後建，提交前更要確認資料完整。
- 貼上表格後看起來有列，但送出數量是 0：通常是因為必填欄位缺值。
- 直接打開 `/auth/data?username=某某` 就想查看別人的數據：query 名稱只影響顯示，不代表權限切換。
