# 修改資料

修改資料不是另一條獨立路由，而是 `/auth` 內部的 `view=modify` 視圖。進來後你會在同一頁裡切換「改用戶名」和「改密碼」兩種模式，並且兩者送出前都一定會經過確認框。

## 你會看到什麼

- 頂部是歡迎文字，會帶出目前登入的用戶名。
- 中間有兩個 tab：`用戶名`、`密碼`。
- 底部固定有返回按鈕，可回到個人概覽。

## 最適合新手的操作順序

1. 先確認自己要改的是名稱還是密碼。
2. 在對應 tab 只填必要欄位。
3. 按送出後，仔細看確認框內容，再決定是否繼續。
4. 成功後等 2 秒左右，頁面會自動回到個人概覽並重新抓一次使用者資料。

## 改用戶名時的實際規則

- 這一頁不會自動預填新用戶名，請自己輸入新的值。
- 前端會先檢查新用戶名長度是否介於 3 到 50 字。
- 如果長度不符，會直接顯示提示，不會送出。
- 送出前一定會先跳確認框，確認你真的要改成該名稱。
- 成功後會切回 `profile`，並重新抓使用者資料，所以概覽上的名稱會同步更新。

## 改密碼時的實際規則

- 必填欄位只有兩個：目前密碼、新密碼。
- 前端會先檢查：
  - 目前密碼不可為空
  - 新密碼至少 6 碼
- 通過後一樣會先跳確認框，再真正送出。
- 成功後也會自動回到個人概覽，並重新同步使用者資料。

## 這一頁有哪些互動細節

- 兩個 tab 只是同頁切換，不是跳轉新頁。
- 改名、改密碼、登入、註冊共用同一組 loading 狀態，所以送出時按鈕會暫時不可按。
- 錯誤與成功訊息會顯示在表單下方；當你切換 auth 的大模式時，這些訊息會被清空。
- 返回按鈕不會離開 `/auth`，而是把視圖切回個人概覽。

## 新手最常見的卡點

- 以為可以直接在這裡看到舊密碼：不會，密碼只能重新輸入。
- 以為按送出就立刻改掉：中間一定還有一次確認框，這是保護機制。
- 改完名稱後一瞬間還沒看到更新：程式會在成功後再抓一次使用者資料，稍等一下即可。
- 以為返回按鈕會回瀏覽器上一頁：這裡其實是回個人概覽，不是做 `router.back()`。
