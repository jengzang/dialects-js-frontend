# 貢獻排行

排行榜不是獨立路由頁，而是登入後個人中心裡的另一個 tab。實作上它仍然掛在 `/auth`，只是把 query `view` 切成 `leaderboard`，再由同一個 ProfileOverview 頁殼把內容換成排行榜面板。

## 你要先知道的事

- 這個畫面要先登入才看得到。
- 它和「概覽」共用同一頁外框，所以看起來像沒換頁，其實是正常設計。
- tab 切換本身會改 query `view`，因此重新整理或回上一頁時，能保留目前是在排行榜還是概覽。

## 最適合新手的操作順序

1. 先從個人中心上方 tab 切到「排行榜」。
2. 先看最上面的兩張摘要卡：在線時長、總查詢次數。
3. 再往下看分類表格，分清楚「分類總排名」與「單一功能排名」。
4. 想回帳號資訊時，再切回「概覽」。

## 這個排行榜實際怎麼載入

- `LeaderboardPanel` 會在元件掛載後自動呼叫 `getLeaderboard()`。
- 載入中會先顯示 loading 畫面。
- 如果請求失敗，畫面上會出現錯誤訊息和「重試」按鈕；按下會再次抓取資料。
- 如果抓取失敗，除了畫面上的錯誤區塊，也會另外跳出 toast 提示。

## 你在畫面上會看到哪些區塊

- 上方標題 `🏆` 排行榜。
- 兩張摘要卡：
  - 在線時長排名
  - 總查詢次數排名
- 下方明細表格：
  - 桌面版會把「分類總排名」和「分類內各功能排名」排在同一列結構裡。
  - 手機版會先顯示分類摘要列，再顯示各功能列。
- 某些指標旁邊有 help icon，滑上去或點擊可看補充說明。

## 表格內容怎麼解讀

- 每個大類都先算一個分類總排名，不是只看單一按鈕使用次數。
- 分類下面還會列出多個實際功能端點的排名。
- `gap` 是你和前一名的差距；如果你已經是第一名，就不會有這個差距值。
- `first place` 會顯示第一名目前的數值，方便你估算還差多少。
- 第一、第二、第三名會用不同徽章樣式標示。

## 新手最容易誤會的地方

- 以為這是全站公開榜單頁：其實它目前是登入後個人中心的一部分。
- 以為切 tab 不會記住：實際上 `view=leaderboard` 會留在網址 query 裡。
- 以為只看總查詢數就夠：下面分類表格才看得出你常用的是哪一類功能。
- 以為重試要回上一頁再進來：其實錯誤區塊上就有重試按鈕。
