這一頁對應 /menu/about/suggestion，用來提交建議與回報問題。

建議新手這樣用：
1. 先判斷你的問題屬於前端介面，還是資料/建置流程。
2. 再按對應卡片前往 GitHub issue。

實際規則：
- 這一頁只有兩張外部連結卡片。
- 第一張會開啟前端專案 issues：dialects-js-frontend。
- 第二張會開啟資料建置專案 issues：dialects-build。
- 兩個連結都會在新分頁開啟，不會離開目前站內路由。

容易踩坑：
- 這頁本身沒有站內表單；點卡片後才是真正提交建議的地方。
- 如果你想回報的是資料生成或字表問題，通常應該走 build 專案，不是前端專案。
