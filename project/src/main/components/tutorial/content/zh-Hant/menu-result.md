# 結果頁

結果頁不是單一固定格式，而是會根據你前一步從哪個查詢分頁進來，自動切換不同結果組件。它同時也是查詢結果與地圖資料的中繼站。

## 你在這一頁會看到什麼

- 頂部會顯示標題，標題來自 `_sourceTab`，有時還會帶上 features。
- 如果目前是查字結果，右上還會多一個下拉，可切換默認 / 調值 / 調類顯示方式。
- 載入中會看到計時器；超過一定時間還會出現長等待提醒。
- 有資料時，頁面會依來源分頁顯示不同結果元件。
- 沒資料時，底部會顯示空態並給你一個「回查詢頁」按鈕。

## 這一頁不是所有模式都長一樣

- 查字（tab1）與查調（tab4）：會走 `CharsAndTones` 類型畫面。
- 查中古（tab2）與查音位（tab3）：會走 `ResultList` 類型畫面。
- 所以你看到的結果區塊不同，不代表錯頁，而是來源模式不同。

## 最適合新手的操作順序

1. 先看標題，確認自己現在看的到底是哪一種結果。
2. 若是查字結果，先決定要看默認、調值還是調類。
3. 接著先看主結果是否真的符合你剛剛輸入的查詢方向。
4. 如果結果很多，先只看幾個最關心的地點或條目。
5. 如果方向不對，直接回原查詢頁改條件；如果方向對，再去地圖頁看空間分布。

## 這一頁有哪些實際規則

- 頁面主要是 watch `globalPayload` 後自動發請求，所以沒有前一步查詢資料時，這裡不會平白出結果。
- 查字、查中古、查音位、查調會分別呼叫不同 API。
- 結果頁不只顯示清單，也會同步生成給地圖用的 mergedData。
- 空態按鈕目前會把你帶回 `/menu/query/zhonggu`，不是回到你剛才所有可能的來源頁。

## 載入時該怎麼判讀

- 有計時器是正常設計，不代表卡死。
- 如果超過 30 秒出現長等待提醒，代表這次查詢可能範圍較大，先等等結果，不要立刻反覆重跑。
- 若你經常等很久，優先考慮縮小地點 / 分區或簡化查詢條件。

## 新手最常見的卡點

- 結果區長得和上一次不一樣：通常只是因為這次來源 tab 不同。
- 明明查字卻看不懂右上角下拉：那是專門切換字音結果的展示方式，不是重新查詢。
- 沒資料：先確認前一步是否真的成功運行，或是不是 globalPayload 已被新的操作覆蓋。
- 想返回卻發現只回查中古：這是目前空態按鈕的固定路徑，不是你操作錯了。

## 什麼時候去別的頁面

- 想改查詢條件：回原本的 query / compare 頁。
- 想看地理分布：去地圖相關頁面。
- 想直接做條件對照：去 compare 類頁面。
