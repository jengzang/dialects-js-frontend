# 註冊

註冊也是 `/auth` 頁中的內部視圖，對應 query `view=register`。這一頁的重點不是表單很多，而是前端其實會先做一輪明確檢查；只要知道哪些欄位會被擋住、成功後會怎麼切回登入，就不容易卡住。

## 你會看到什麼

- 標題旁邊同樣有 🎁 權益按鈕，可先看會員權益說明。
- 表單固定有四格：用戶名、郵箱、密碼、確認密碼。
- 底部有「已有帳號？」連結，可直接回登入視圖。

## 最適合新手的操作順序

1. 先想好用戶名，再填郵箱。
2. 輸入密碼後，立刻在確認密碼再輸一次。
3. 按送出後先等結果，不要連按。
4. 成功後頁面會自動切回登入視圖，再用剛建立的帳號登入。

## 這一頁有哪些實際規則

- auth 頁內部由 query `view` 控制；按連結切換登入/註冊時，實際上是在改 `view`。
- 註冊按鈕在 loading 中會被停用。
- 前端送出前會依序檢查：
  - 用戶名長度是否介於 3 到 50 字
  - 郵箱格式是否正確，且網域必須在允許清單內，或符合 `.edu.cn`、`.edu`、`.ac.uk`、`.ac.jp` 這類教育網域
  - 密碼是否至少 6 碼
  - 兩次密碼是否完全一致
- 只要前面任何一步不通過，就不會送到後端。

## 註冊成功後會發生什麼

- 成功訊息會先顯示約 1 秒。
- 接著系統會自動把 `view` 切回 `login`。
- 切回登入頁時，也會一起清掉目前的錯誤與成功訊息。

## 常見提示該怎麼理解

- 提示用戶名長度錯誤：通常是太短，或前後空白去掉後實際長度不夠。
- 提示郵箱無效：不一定只是格式錯，也可能是網域不在前端白名單內。
- 提示兩次密碼不一致：請直接重填確認密碼，不要只靠肉眼檢查。
- 提示用戶名已存在或郵箱已存在：這兩種錯誤會被明確轉成對應提示，直接改另一個值即可。

## 新手最常見的卡點

- 用學校或公司郵箱卻一直過不了：先確認是不是允許的網域類型。
- 以為註冊成功會自動登入：目前不會，成功後是自動切回登入頁。
- 在 loading 時重複點送出：按鈕其實已被禁用，等結果最省事。
- 先看權益彈窗後找不到註冊入口：彈窗可直接切去註冊，或回到表單照常填寫。
