# 比較中古

把兩組中古條件放在一起，直接看它們在各地的對應結果是否相近、部分相近或差異明顯。這一頁是真正的「兩組條件對打」。

## 你在這一頁會看到什麼

- 上方只有一套「目前正在編輯的條件」。
- 中間有兩個已選區：組 1、組 2。
- 你不是直接同時編兩套條件，而是先在中央編好一套，再用「加入」按鈕送到組 1 或組 2。
- 下方同樣有地點 / 分區設定。
- 按下比較後會直接跳到地圖頁，不會先去結果頁。

## 一定要先理解的核心交互

- 中央區是「條件編輯區」。
- 左右兩個已選框是「待比較條件集合」。
- 你每按一次「加入組 1」或「加入組 2」，其實是在把當前條件做成一個快照，存進對應組裡。
- 所以如果你改了中央條件，舊的已加入項目不會自動跟著變。

## 最適合新手的操作順序

1. 先決定比較角度：聲母、韻母或聲調。
2. 在中央區選鍵名、鍵值，組出第一條有效中古條件。
3. 確認下方條件展示沒問題後，按「加入組 1」。
4. 再修改中央條件，組出第二條條件，按「加入組 2」。
5. 如果同一組想放多條件，可以繼續往同一組追加。
6. 最後補地點 / 分區，按下比較進地圖頁。

## 這一頁有哪些實際規則

- 只有當前條件至少選了有效鍵名，而且其中至少一個鍵有實際值時，「加入」按鈕才可用。
- 組 1 和組 2 都至少要各有一條條件，最終比較按鈕才會可用。
- 同一組裡所有條件的「排除」設定必須一致；如果你想加入的新條件排除設定不同，頁面會警告並拒絕加入。
- 比較時會把每組所有已加入條件的 combinations 合併後送出，所以一組可以代表多條件聯合觀察。

## 新手怎麼避免加錯組

- 第一次做時，建議先只做「組 1 一條、組 2 一條」，確認流程後再增加。
- 加入後先看已選列表中的文字摘要，有沒有把鍵和值放對。
- 如果加錯了，可直接點該項右側的刪除按鈕移除，再重新加入。

## 地圖頁上會看到什麼

- 地圖圖例會顯示相同、高度相似、部分相似、不同、未知等類型。
- 這代表它不是單純看有沒有，而是在比較兩組條件於各地的接近程度。
- 若你還沒把單組條件理解清楚，建議先回「查中古」各自查一遍，再回來比較。

## 新手最常見的卡點

- 「加入」按鈕灰掉：表示當前中央條件還沒組完整。
- 可以加入，但最後比較按鈕還是灰的：通常是組 1 或組 2 其中一邊還是空的。
- 加入時被警告：多半是同一組前後使用了不同排除設定。
- 地圖結果太難懂：先讓每組只保留一條條件，別一開始就把多條件混在一起。

## 什麼時候改用別的頁面

- 想先查單組中古條件：改用「查中古」。
- 想比較兩個字：改用「比較漢字」。
- 想比較兩個調類是否合流：改用「比較調類」。
