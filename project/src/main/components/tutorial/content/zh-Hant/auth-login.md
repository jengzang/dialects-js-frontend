# 登錄

這一頁不只是輸入帳號密碼而已，實際上是 `/auth` 頁面的其中一個內部視圖，並且會用 query `view=login` 來維持目前狀態。對新手來說，先弄清楚「登入方式切換」「成功後會去哪裡」「按鈕什麼時候不能按」最重要。

## 你會看到什麼

- 標題旁邊有一個 🎁 按鈕，按下會打開會員權益彈窗。
- 中間有兩個切換按鈕：`郵箱登錄`、`用戶名登錄`。
- 下方會依你目前選的模式，只顯示對應欄位：
  - 郵箱登錄：郵箱 + 密碼
  - 用戶名登錄：用戶名 + 密碼
- 最下面有「沒有帳號？」的連結，可直接切到註冊視圖。

## 最適合新手的操作順序

1. 先確認自己要用郵箱還是用戶名登入。
2. 再輸入對應帳號欄位與密碼。
3. 按登入按鈕後，先等載入狀態結束，不要連點。
4. 若成功，系統會先同步使用者資料，再跳轉或刷新頁面。

## 這一頁有哪些實際規則

- auth 頁內部由 query `view` 控制，目前這一篇對應的是 `view=login`。
- 登入方式切換不是開新頁，而是同一個表單內切換欄位。
- 切換「郵箱 / 用戶名」時，密碼欄會被清空；這是程式刻意設計，不是輸入消失故障。
- 登入、註冊、改名、改密碼共用同一組 loading 狀態，所以送出後按鈕會暫時不可按。
- 前端在送出前只會先檢查密碼長度是否至少 6 碼；帳號格式錯誤或帳密不符，多半要看後端回傳提示。

## 登入成功後會去哪裡

- 如果你原本是從受保護頁面被導來登入，且網址裡有安全的 `redirect` 參數，成功後會回到原頁。
- 如果沒有可用的 `redirect`，系統會直接重新整理頁面，讓整站登入狀態同步完成。
- 程式只接受站內相對路徑作為 `redirect`；像外站網址，或再跳回 `/auth` 自己，都会被忽略。

## 失敗時先檢查哪幾件事

- 選錯登入模式：你拿郵箱去填用戶名模式，或反過來。
- 密碼少於 6 碼：前端會直接擋下。
- 帳密不符：程式會把 `Invalid credentials` 類型錯誤轉成較易懂的登入失敗提示。
- 剛切換登入模式後忘了重新輸入密碼：這是最常見的小卡點之一。

## 新手最常見的誤解

- 以為切換登入模式後兩邊內容會共用：其實只有帳號欄各自保留，密碼會清空。
- 以為登入成功卻沒立刻跳畫面是故障：實際上程式還會先抓使用者資料、角色，並初始化在線時長追蹤。
- 以為 🎁 按鈕會直接註冊：它只會開權益彈窗；真的要註冊，要再從彈窗或底部連結切過去。
