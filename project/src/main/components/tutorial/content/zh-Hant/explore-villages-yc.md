# 陽春村落

這一頁專門顯示陽春範圍的村落樹狀資料。它不像廣東自然村樹狀圖那樣按需向 API 分市載入；這一頁是在前端直接讀取 `yc_villages.json`，把 JSON 內容整理成可展開的樹。

## 這一頁有什麼

- 頁首標題。
- 一行資料來源說明文字。
- 一個搜尋框。
- 下方是一整棵可展開、可收合的村落樹。

這一頁沒有表格、沒有地圖按鈕，也沒有逐市載入按鈕。

## 資料結構怎麼看

資料來自本地 JSON，最上層直接是陽春各鎮街，例如 JSON 一開始就有「春城街道」。

元件會把 JSON 整理成樹狀節點，大致可理解為：
- 鎮／街道
- 類別或村委層
- 更下層分組
- 最底層村名

如果某一層是字串陣列，陣列中的每個字串都會變成最底層葉節點。

## 這頁特別做了什麼清理

在把 JSON 轉成樹時，元件會主動跳過空內容：
- 空陣列不顯示。
- 空物件不顯示。

所以如果你發現某些分類在 JSON 裡存在、但畫面上沒看到，通常表示那個分類底下沒有實際內容，系統已經自動省略。

## 建議操作順序

1. 先從最上層鎮街往下展開。
2. 需要時再用搜尋框找村名或上層分類名。
3. 搜到結果後，沿著自動展開的路徑往下看。
4. 若想看某個命中節點下的完整內容，可再手動展開該節點。

## 搜尋怎麼運作

搜尋是以前端遞迴方式比對節點名稱。

具體行為如下：
- 若某個深層子節點符合關鍵字，它上面的路徑節點會保留下來，而且自動展開。
- 若是父節點自己符合關鍵字，但子節點沒有被關鍵字命中，該父節點仍會保留，但預設不自動展開。
- 這時你仍可手動按 ＋ 查看它原本的全部下層內容。
- 沒有輸入搜尋字時，會顯示完整樹。
- 若完全沒有匹配結果，頁面會顯示空結果提示。

## 樹狀節點怎麼操作

- 📁 代表還有子節點。
- 📍 代表最底層節點。
- 點節點列本身即可展開或收合。
- 有子節點時，右側會出現 ＋ 按鈕；展開後按鈕會旋轉。
- 搜尋命中的文字會高亮。

## 你會看到哪些輸出

- 初始進頁後，會直接看到整理好的樹，不需要再另外載入資料。
- 展開節點後，會看到下層分類或村名。
- 搜尋時，只保留符合條件的節點與必要路徑。
- 若結果為空，頁面會顯示沒有結果。

## 和廣東自然村樹狀圖的差別

這一頁和廣東自然村樹狀圖很像，但用途不同：
- 陽春頁面資料是本地 JSON，開頁即可用。
- 這頁沒有地圖彈窗。
- 這頁不顯示經緯度與方言分布。
- 這頁更適合專注閱讀陽春內部的村落層級。

## 常見容易誤會的地方

- 看不到某個分類，不一定是漏資料；可能是它在 JSON 中本來就是空陣列或空物件，已被自動隱藏。
- 搜尋命中父節點時，系統不一定會自動把它全部下層打開；想看完整內容要自己再展開。
- 這頁不能直接定位地圖，因為元件只做樹狀瀏覽。
- 這頁顯示的是陽春本地資料，不適合拿來代表整個廣東省自然村資料庫。

## 適合什麼情況使用

- 想專看陽春範圍的村落層級。
- 想快速確認某個鎮街、村委或村名是否出現在陽春資料中。
- 想在不切換到地圖或表格的情況下，直接做結構化閱讀。
