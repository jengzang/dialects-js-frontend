# 自定義分區

「自定義分區」是獨立路由 `/auth/regions`，通常從個人中心的「我的分區」按鈕進來。這頁的核心不是只有新增分區，而是同時支援搜尋、編輯、刪除，以及用地點選擇器或手動輸入來組成分區。

## 進頁前先知道的事

- 頁首顯示的用戶名來自 query `username`，主要用於展示。
- 頁面一掛載就會自動呼叫 `loadRegions()` 載入目前分區。
- 搜尋只會過濾分區名稱，不會搜尋描述或內含地點。

## 你會看到什麼

- 頂部有返回按鈕、標題與用戶名 badge。
- 中間有兩個統計：分區數量、所有分區合計的不重複地點數。
- 工具列有新增、重新整理與搜尋框。
- 下方是分區卡片列表；每張卡片可編輯或刪除。

## 最適合新手的操作順序

1. 先等列表載入完成，確認你目前已有沒有分區。
2. 沒有分區時，直接按空狀態或工具列裡的新增。
3. 建立分區時，先填名稱，再決定用地點選擇器還是手動輸入地點。
4. 存檔前先看儲存按鈕是否可按，以及地點數是否合理。
5. 建好後如果要微調，再從卡片右上角鉛筆按鈕進編輯。

## 新增與編輯有哪些實際規則

- 按新增會打開編輯彈窗；如果分區樹資料還沒載入，系統會順手抓一次分區資料。
- 編輯既有分區時，`region_name` 欄位會被鎖住不能改；這是 popup 元件明確設定的禁用狀態。
- 只有同時滿足以下條件，儲存按鈕才可按：
  - 分區名稱不是空白
  - 至少有 1 個地點
  - 地點數不超過 `CUSTOM_REGION_MAX_LOCATIONS`，目前上限是 1000
- 儲存中按鈕會進入 loading 狀態，避免重複送出。

## 地點怎麼加入分區

你有兩種方式，可以混著用：

- 用「選擇地點」按鈕打開分區/地圖選擇器。
- 直接在文字框手動輸入地點。

### 手動輸入時的規則

- 文字框會用空白字元切開內容，所以空格、換行都能分隔地點。
- 系統會自動去重複，最後 `editingRegion.locations` 只保留唯一值。
- 下方會即時顯示兩個計數：
  - `M`：手動輸入、但不在分區樹可用地點集合中的數量
  - `T`：可在分區樹中辨識到的地點數量

### 用選擇器時的規則

- 第一次開選擇器前，如果分區資料尚未快取，頁面會先抓取資料。
- 選完地點後，結果會回寫到文字框與已選標籤區。
- 如果你一次選超過 1000 個地點，會直接跳警告，且不會套用這次選擇。

## 重名與刪除的特別情況

- 新增分區時，如果名稱和既有分區重複，系統會先跳確認框，再決定是否繼續送出。
- 刪除分區前一定會先跳確認框。
- 某張卡片正在刪除時，垃圾桶按鈕會禁用並顯示 loading 圖示。

## 卡片列表怎麼看

- 每張卡片會顯示分區名稱、地點數、建立日期、描述。
- 卡片裡最多先展示前 10 個地點；超過時會顯示 `+N`，代表還有更多沒有全部攤開。
- 如果搜尋後沒有結果，會顯示無符合分區；若完全沒有分區，則顯示建立第一個分區的空狀態。

## 新手最常見的卡點

- 以為編輯時可以順便改分區名稱：目前既有分區名稱會被鎖住。
- 以為地點要用逗號分隔：其實這裡是用空白或換行拆分。
- 以為輸入重複地點會報錯：系統會自動去重。
- 以為超過上限還是能先存草稿：實際上超過 1000 個地點時，儲存會被擋住。
- 以為搜尋能找描述內容：目前只比對分區名稱。
