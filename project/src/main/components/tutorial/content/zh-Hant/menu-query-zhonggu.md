# 查中古

用中古條件去反查各地讀音資料。這一頁不是直接輸入漢字，而是先組裝「你要找哪一類中古音」，再看它在不同地點對應成什麼樣子。

![查中古頁面示意](/tutorial/menu-query-zhonggu/overview.svg)

## 你在這一頁會看到什麼

- 第一行先選大類：聲母、韻母、聲調。
- 旁邊有「排除」下拉，可過濾多地位標記等情況。
- 中間先選鍵名，再選鍵值，最後由下方條件組件組成真正要查的中古條件。
- 下方仍然是共用的地點 / 分區設定。
- 按下運行後，頁面會跳到結果頁，不是留在本頁顯示結果。

## 最適合新手的操作順序

1. 先決定你要查的是聲母、韻母還是聲調。
2. 再點選一個或多個鍵名，例如攝、等、開合之類，縮小查詢方向。
3. 在對應的鍵值區把值選完整，讓下方條件組件真正組出可查的條件。
4. 如果資料太雜，再打開「排除」下拉加入過濾條件。
5. 最後再補地點或分區，然後按運行去結果頁看輸出。

## 這一頁有哪些實際規則

- 不是只選了卡片就能查；必須讓下方中古條件組件形成有效條件後，按鈕才會可用。
- 鍵名與鍵值是聯動的；換了字表或換了鍵名後，先前不合法的值會被清掉。
- 「排除」選項會進入真正請求條件中，不只是畫面上的備註。
- 目前查出的結果會帶著 `_sourceTab: tab2` 進結果頁，所以結果頁會依照「查中古」模式解析。

## 新手該怎麼理解這個頁面

- 可以把它想成三步：先選分析角度，再選條件，再決定查哪些地點。
- 如果你還不熟中古術語，第一次不要同時選太多鍵名；從 1 到 2 個核心條件開始最容易看懂。
- 如果你只是想確認某個字怎麼讀，不要硬用這頁，直接去查字會更直觀。

## 地點 / 分區怎麼配合使用

- 當中古條件比較寬時，結果很容易很多，所以分區特別有用。
- 想看整體分布，可以先只選分區；想核對特定點，可以直接輸入地點。
- 底部顯示的地點數就是這次實際會落入查詢範圍的數量，數量太大時建議先收窄。

## 運行後你會得到什麼

- 系統會跳到結果頁，由結果頁再根據這組中古條件發請求並呈現資料。
- 如果你後續想做更精細比較，可以先用這頁摸清條件，再去「比較中古」把兩組條件並列。

## 新手最常見的卡點

- 按鈕是灰的：通常表示下方中古條件其實還沒組成有效查詢，不是壞掉。
- 結果過多：先減少鍵名數量，或直接加分區。
- 結果過少：可能是條件太嚴，先去掉一兩個鍵值再試。
- 看不懂排除效果：先在不排除的情況下跑一次，再加排除重跑，比較容易理解差別。

## 什麼時候改用別的頁面

- 想從字形出發：改用「查字」。
- 想從實際音值倒推中古來源：改用「查音位」。
- 想把兩組中古條件直接對打：改用「比較中古」。
